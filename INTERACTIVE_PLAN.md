# 交互式 Dashboard 实现方案

## 🎯 目标需求回顾

1. 清晰展示 10+ 个会话（卡片式布局）
2. Dashboard 上直接提问，发送到选中的会话
3. 显示最近 1-2 轮问答
4. 点击查看完整对话历史

## ⚠️ 技术限制

### 核心问题
Claude Code CLI 是独立的交互式进程，Dashboard 无法直接向其发送输入，因为：
- Claude 进程直接绑定到终端 TTY
- 没有 IPC（进程间通信）机制
- 不支持远程控制 API

### 可行的解决方案

## 方案 A：只读监控 + 手动切换（推荐）⭐⭐⭐⭐⭐

**实现方式**：
- Dashboard 展示所有会话和最近对话
- 点击会话卡片 → 显示终端路径和 PID
- 用户手动切换到对应终端窗口
- Dashboard 实时显示该会话的最新输出

**优点**：
- 实现简单，无需修改 Claude Code
- 稳定可靠
- 符合当前架构

**缺点**：
- 需要手动切换终端

**适用场景**：当前最实际的方案

---

## 方案 B：Tmux 集成（需要重启会话）⭐⭐⭐⭐

**实现方式**：
1. 所有 Claude 会话在 tmux 中启动
2. Dashboard 通过 `tmux send-keys` 发送命令
3. Dashboard 通过 `tmux capture-pane` 捕获输出

**优点**：
- 可以真正实现远程控制
- 可以发送输入到会话

**缺点**：
- 需要重新启动所有会话
- 需要学习 tmux 使用

**实现示例**：
```bash
# 启动 Claude 会话
tmux new-session -d -s claude-session-1 "cd /path/to/project && claude"

# 从 Dashboard 发送命令
tmux send-keys -t claude-session-1 "你的问题" Enter

# 捕获输出
tmux capture-pane -t claude-session-1 -p
```

---

## 方案 C：代理模式（需要开发新工具）⭐⭐⭐

**实现方式**：
1. 开发一个 Claude 代理工具
2. 代理工具启动 Claude 进程并捕获 I/O
3. Dashboard 通过代理与 Claude 通信

**优点**：
- 完全控制
- 可以添加更多功能

**缺点**：
- 开发复杂度高
- 需要维护代理工具

---

## 方案 D：混合方案（推荐实现）⭐⭐⭐⭐⭐

**实现方式**：
1. **现有会话**：只读监控 + 快速跳转
2. **新会话**：从 Dashboard 启动（在 tmux 中）
3. **统一界面**：两种会话在 Dashboard 中统一展示

**功能设计**：

### 1. 会话卡片（支持 10+ 个）

│ [查看详情] [跳转终端] [新建会话]     │
└─────────────────────────────────────┘

```

### 3. 详情页（点击展开）
```
