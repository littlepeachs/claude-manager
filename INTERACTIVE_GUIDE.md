# 🎉 交互式 Claude Code Dashboard 使用指南

## ✨ 新功能概览

Dashboard 已升级为**交互式管理面板**，支持：

1. ✅ **卡片式布局** - 清晰展示 10+ 个会话
2. ✅ **直接提问** - 在 Dashboard 输入问题，发送到选中的会话
3. ✅ **最近对话预览** - 每个卡片显示最新讨论主题
4. ✅ **完整历史查看** - 点击查看详情查看所有对话
5. ✅ **Tmux 集成** - 支持远程控制 tmux 会话
6. ✅ **会话管理** - 创建、终止会话
7. ✅ **紫白配色** - 优雅的紫色主题

---

## 🚀 快速开始

### 访问 Dashboard

打开浏览器访问: **http://localhost:8765**

你将看到：
- 顶部输入框（选中会话后可用）
- 总览统计卡片
- 会话卡片网格
- 右下角创建按钮

---

## 📋 界面说明

### 1. 输入区域（顶部）

```
┌─────────────────────────────────────────┐
│ 当前激活: claude_tool (可远程控制)      │
│ ┌─────────────────────────────────────┐ │
│ │ 输入你的问题...                      │ │
│ └─────────────────────────────────────┘ │
│ [发送]                                  │
└─────────────────────────────────────────┘
```

- 显示当前选中的会话
- **可远程控制** = tmux 会话，可直接发送消息
- **只读模式** = 普通会话，需要手动切换终端

### 2. 会话卡片

每个卡片显示：

```
┌─────────────────────────────────────┐
│ claude_tool              [TMUX]     │
│ PID 88378  ⏱️ 45m                   │
│ /Users/liwentao/.../claude_tool     │
├─────────────────────────────────────┤
│ 💬 我希望干一件这样的事情...         │
├─────────────────────────────────────┤
│ Token: 50K | 内存: 331MB | CPU: 12% │
├─────────────────────────────────────┤
│ [查看详情] [终止]                    │
└─────────────────────────────────────┘
```

- **TMUX** 标签 = 可远程控制
- **NORMAL** 标签 = 只读模式
- 点击卡片 = 激活该会话
- 蓝色边框 = 当前激活的会话

## 🎯 核心功能

### 功能 1: 向会话提问

**适用于**: Tmux 会话（标记为 TMUX）

**步骤**:
1. 点击一个 tmux 会话卡片（激活它）
2. 在顶部输入框输入问题
3. 点击"发送"或按 Enter
4. 消息会自动发送到该会话
5. 等待几秒，卡片会更新显示回复

**注意**: 
- 只有 tmux 会话支持远程发送
- 普通会话显示"只读模式"，无法远程控制

---

### 功能 2: 创建新的 Tmux 会话

**步骤**:
1. 点击右下角的 **+** 按钮
2. 输入项目路径（例如: `/Users/liwentao/Desktop/Research/claude/test`）
3. 点击确定
4. 新会话会自动创建并显示在卡片中

**创建的会话**:
- 自动在 tmux 中启动
- 支持远程控制
- 可以直接从 Dashboard 提问

---

### 功能 3: 查看对话历史

**步骤**:
1. 点击会话卡片上的"查看详情"按钮
2. 弹出模态框显示完整对话历史
3. 查看所有问答对话
4. 点击"关闭"返回

**显示内容**:
- 所有历史问答
- 问题和回答的完整内容
- 时间顺序排列

---

### 功能 4: 终止会话

**步骤**:
1. 点击会话卡片上的"终止"按钮
2. 确认操作
3. 会话进程会被终止
4. 卡片自动从列表中移除

**注意**: 
- 终止操作不可恢复
- 会话数据会保留在 `~/.claude/` 目录

---

## 🔄 两种会话类型对比

### Tmux 会话（推荐）

**特点**:
- ✅ 可以从 Dashboard 直接提问
- ✅ 支持远程控制
- ✅ 可以捕获实时输出
- ✅ 标记为 [TMUX]

**创建方式**:
- 在 Dashboard 点击 + 按钮创建
- 或手动: `tmux new-session -s my-session "cd /path && claude"`

**使用场景**:
- 需要从 Dashboard 控制的会话
- 多任务并行处理
- 远程监控和管理

---

### 普通会话（现有会话）

**特点**:
- ⚠️ 只读模式，无法远程发送消息
- ✅ 可以查看状态和历史
- ✅ 标记为 [NORMAL]

**来源**:
- 直接在终端运行 `claude` 命令
- 现有的所有会话

**使用场景**:
- 监控现有会话
- 查看 token 消耗
- 快速定位会话位置

## 💡 使用技巧

1. 创建多个 tmux 会话用于不同项目

- **快速发送**: 输入后按 Enter
- **查看详情**: 点击"查看详情"
- **创建会话**: 点击右下角 +

## 🔧 高级功能

### 手动创建 Tmux 会话

如果你想在终端手动创建可控制的会话：

```bash
# 创建新的 tmux 会话
tmux new-session -s my-project "cd /path/to/project && claude"

# 分离会话（返回终端）
# 按 Ctrl+B 然后按 D

# 会话会自动出现在 Dashboard 中
```

### 重新连接 Tmux 会话

```bash
# 列出所有 tmux 会话
tmux list-sessions

# 连接到指定会话
tmux attach-session -t my-project
```

### API 使用

Dashboard 提供 REST API：

```bash
# 获取所有会话
curl http://localhost:8765/api/sessions

# 发送消息
curl -X POST http://localhost:8765/api/sessions/SESSION_ID/send \
  -H "Content-Type: application/json" \
  -d '{"session_id": "SESSION_ID", "message": "你好"}'

# 创建会话
curl -X POST http://localhost:8765/api/sessions/create \
  -H "Content-Type: application/json" \
  -d '{"project_path": "/path/to/project"}'

# 终止会话
curl -X DELETE http://localhost:8765/api/sessions/SESSION_ID
```

## ⚠️ 注意事项

### 1. Tmux 会话的限制

- Token 统计可能不准确（因为无法读取 ~/.claude/ 数据）
- 需要手动管理会话生命周期
- 会话名称自动生成（claude-timestamp）

### 2. 普通会话的限制

- 无法从 Dashboard 发送消息
- 只能查看状态，不能控制
- 需要手动切换到终端操作

### 3. 性能考虑

- Dashboard 每秒刷新一次
- 大量会话（10+）可能影响性能
- 建议及时终止不需要的会话

## 🐛 故障排除

### 问题 1: 无法发送消息

**原因**: 选中的是普通会话（NORMAL）

**解决**: 
- 创建新的 tmux 会话
- 或在终端手动操作该会话

### 问题 2: 会话列表为空

**检查**:
1. 是否有 Claude 进程在运行: `ps aux | grep claude`
2. Dashboard 是否正常运行: `curl http://localhost:8765/health`
3. 查看 Dashboard 日志

### 问题 3: 创建会话失败

**可能原因**:
- 项目路径不存在
- 没有权限
- tmux 未安装

**解决**:
```bash
# 检查 tmux
which tmux

# 检查路径
ls -la /path/to/project

# 手动测试
tmux new-session -s test "cd /path && claude"
```
